## 🔄 ТЕКУЩАЯ РАБОТА

### Фаза 4.8.1: Исправление Architect — пространственное мышление AI
**Статус:** спроектировано, не реализовано. Начать с выбора варианта.

#### Проблема (выявлена при тестировании)
Architect (AI-мозг) не понимает физические ограничения слайда:
1. **Overflow** — контент вылезает за 720px (буллеты начинаются на y=503, заканчиваются на y=821)
2. **Нет креативного редизайна** — Architect копирует структуру оригинала вместо трансформации (длинные буллеты остаются буллетами, хотя должны стать карточками)
3. **Таблица: неоптимальное распределение ширин колонок** — длинный заголовок 3-й колонки получает всю ширину, средняя сжимается

#### Корень проблемы
Layout Engine — исполнитель. Он НЕ МОЖЕТ решить проблему переполнения — это задача AI-мозга.
Architect должен:
- Понимать бюджет пространства (680px рабочей высоты)
- Прикидывать сколько текста влезет
- Трансформировать контент (длинные буллеты → карточки, таблица 10 строк → top-5)
- Если ошибся — получить фидбек от Layout Engine и исправиться (max 2 retry)

#### Открытый вопрос: Вариант A или B?

**Вариант A — Вернуть двухпроходную систему (как задумано в handoff.md §4.4):**
- Pass 1: Classifier (flash) → JSON с objects, overload, series_context
- Pass 2: Generator (flash) → LayoutPlan, получает ТОЛЬКО нужные модули (по modules_map.md)
- Плюсы: Generator получает чистый сфокусированный контекст, лучше следует правилам
- Минусы: +1 LLM вызов на слайд, сложнее код

**Вариант B — Один вызов, переписать промпт Architect:**
- Добавить бюджет пространства, трансформационные правила, chain-of-thought
- Плюсы: дешевле, проще
- Минусы: контекст остаётся большим (все модули), AI может "тонуть" в правилах

**В обоих вариантах нужно:**
- Retry-механизм: Layout Engine проверяет overflow → формирует ошибку → Orchestrator передаёт Architect'у → max 2 итерации
- Бюджет пространства в промпте (heading ~80px, строка таблицы ~30px, буллет ~22px/строка и т.д.)
- Трансформационное мышление (правила трансформации контента)

#### Что не используется из config/
- `config/classifier.md` — полноценный промпт классификатора, НЕ подключён к коду
- `config/modules_map.md` — таблица фильтрации модулей, НЕ используется в prompt_assembler
- `prompt_assembler.py` грузит ВСЕ модули всегда, вместо фильтрации по modules_map

#### Решение принимает Алишер в начале следующего чата.

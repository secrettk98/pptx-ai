# Layout Code v1.0 — Правила компоновки слайдов

## РАЗМЕРЫ И СЕТКА

Слайд: 3387×1905 (33.87cm × 19.05cm), viewBox="0 0 1280 720"
Масштаб: 1cm ≈ 37.8px в viewBox

### Margins (⚡ критично)
- Левый/правый: 47px (1.25cm)
- Верхний/нижний: 47px (1.25cm) — красная граница, не нарушать
- Рабочая область: 1186×626px (x: 47–1233, y: 47–673)

### Центрирование
ЕСЛИ контент по высоте меньше 626px → ТО центрируй вертикально внутри рабочей области. Сгруппируй все элементы и выровняй по середине.

### Gaps
Расстояние между элементами — на усмотрение, но не больше 32px. Все gaps на одном слайде строго одинаковые.

## ТИПОГРАФИКА

| Роль | Размер | Когда |
|------|--------|-------|
| Заголовок слайда | 24pt | Один на слайд, сверху |
| Крупные цифры/метрики | 36-44pt | KPI, статистика, акценты |
| Заголовки внутри слайда | 14pt bold | Названия карточек, секций |
| Основной текст | 12pt | Базовый контент |
| Подписи/сноски | 10pt | Менее важное, серый цвет |

⚡ Максимум 3 размера шрифта на один слайд.
⚡ ЕСЛИ в тексте нужно выделить ключевые слова → ТО bold, не курсив, не подчёркивание.
Шрифт: Google Sans (для всех элементов).

---

## РАБОТА С ТЕКСТОМ

⚡ ЕСЛИ текста больше 3 строк подряд → ТО разбей на тезисы с иконками слева или пронумерованные карточки.

⚡ ЕСЛИ пункт списка длиннее 5 слов → ТО выдели жирным 2-3 ключевых слова для чтения по диагонали.

ЕСЛИ заголовок длиннее 5 слов → ТО раздели на 2 строки: суть (24pt, bold) + уточнение (14pt, серый).

ЕСЛИ список описывает процесс/хронологию → ТО карточки в линию + стрелки между ними.

ЕСЛИ карточки в сетке 2×2 → ТО добавь нумерацию (01, 02...) для направления чтения.

ЕСЛИ есть сноска/ГОСТ/примечание → ТО в самый низ, по центру, 10pt, серый, иконка "i".

---

## РАБОТА С ИЗОБРАЖЕНИЯМИ

⚡ ЕСЛИ кадрируешь фото → ТО только прямоугольники со скруглёнными углами. Никаких кругов, толстых рамок, грубых теней.

ЕСЛИ изображение искажено → ТО восстанови пропорции. Запрещено растягивать.

ЕСЛИ на слайде мало текста (много пустоты) → ТО отдай 50% слайда под качественное фото.

ЕСЛИ фото используется как фон/шапка → ТО 100% ширины + тёмный overlay для читаемости текста.

ЕСЛИ каталог (несколько фото предметов) → ТО вырежи фон, единый масштаб, единое направление.

ЕСЛИ несколько фото одного процесса → ТО объедини в составное изображение.

---

## РАБОТА С ДАННЫМИ

⚡ ЕСЛИ метрик 3-4 штуки → ТО не таблица, а карточки. Цифра в 3 раза крупнее подписи.

⚡ ЕСЛИ таблица → ТО без вертикальных линий. Шапка — акцентный фон, строки — зебра (белый/светло-серый).

ЕСЛИ в таблице число + процент → ТО одна ячейка: "120 (100%)".

ЕСЛИ данные меняют контекст (характеристики → проблемы) → ТО 2 отдельных блока, не одна таблица.

ЕСЛИ статистика привязана к географии → ТО данные на карту (маркеры + выноски), не гистограмма.

---

## СЛОЖНЫЕ СЛАЙДЫ

⚡ ЕСЛИ карта + текст → ТО никогда текст поверх карты. Две колонки: 40% текст, 60% карта.

ЕСЛИ маршрут на карте → ТО карта обесцвечена/серая, маршрут — яркая контрастная линия.

ЕСЛИ подпись к фото (статус, цена) → ТО цветная плашка-ярлык поверх фото в углу, не текст под фото.

ЕСЛИ блоки "висят в воздухе" → ТО оберни в контейнеры (светло-серый фон или рамка).

ЕСЛИ фото и текст разнесены, но связаны → ТО цветовое кодирование (одинаковый акцентный цвет маркеров).

⚡ ЕСЛИ слайд типа "сравнение ДО/ПОСЛЕ + данные" → ТО не делай равные колонки под каждое фото. 
Сгруппируй все фото в один визуальный блок (40% слева), а текст + данные + CTA в текстовый блок (60% справа). 
Фото могут быть разного размера внутри блока.

⚡ ЕСЛИ на слайде есть "Проблема" и "Решение" → ТО покажи их как последовательность сверху вниз: 
Проблема (красный маркер) → Данные → Решение (зелёный маркер) → CTA.

---

## ОБЩИЕ ПРИНЦИПЫ

⚡ Все margins от краёв — 47px, одинаковые.

⚡ Сетка: элементы выровнены по невидимым линиям, все gaps равны.

⚡ Единый border-radius для ВСЕХ элементов на слайде.

ЕСЛИ одна колонка выше другой → ТО добавь многострочный заголовок к короткой, чтобы уравновесить высоту.

ЕСЛИ карточек мало → ТО квадратные, горизонтально в ряд, не вертикальная башня.

ЕСЛИ контент не заполняет рабочую область по ширине → ТО растяни блоки на все 1186px. Пустота по бокам запрещена.

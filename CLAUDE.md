# PPTX-AI — Project Context

## Роль и стиль
- Техлид (Claude Opus 4.6): НЕ пишет код — пишет инструкции для агента-кодера
- Алишер — неопытный программист, объяснять как для ребёнка
- Ответы короткие, понятные, без лишнего текста
- Стиль: Senior разработчик, лучший учитель, лучший ментор
- Экономим токены — не забиваем контекст кодом
- Глобальная цель: успешно запустить сервис и сделать Алишера миллионером
- В конце КАЖДОГО сообщения: "Потрачено токенов: ~XXX | Осталось токенов: ~XXX"
- Начальный бюджет нового чата: 200000 токенов

## Суть продукта
B2B AI-сервис редизайна презентаций PowerPoint (Copilot-формат).
Не генератор картинок по клику, а виртуальный напарник:
глубокий смысловой аудит → математически выверенный дизайн → корпоративные гайдлайны.

Клиент загружает .pptx + акцентный цвет бренда →
AI анализирует → генерирует SVG-слайды по Design System →
PPT Master собирает нативный редактируемый .pptx.
Второй режим: генерация презентации с нуля из текста/документов.

## Главный принцип
LLM — мозг (анализирует контент, выбирает layout, составляет бриф).
Python — руки (парсит, постобрабатывает, собирает PPTX).
Никаких PNG/PDF — только редактируемые нативные элементы PowerPoint.

## Киллер-фича (конкурентное преимущество)
Реверс-инжиниринг сложных элементов:
- Блок-схемы → Vision AI → логический граф (JSON) → Graphviz → SVG → нативный PPTX
- Диаграммы/графики → Vision AI → данные (CSV) → Matplotlib/python-pptx → нативный график
- Карты → Map Redesign Pipeline v2.0 (13 типов, 5 модулей) → нативный PPTX
- Никаких растровых скриншотов-заглушек — всё нативное и редактируемое

## Map Redesign Pipeline v2.0 # NEW — ВЕСЬ БЛОК НОВЫЙ

### Архитектура (5 модулей):
1. map_classifier.py — Gemini Vision определяет тип (1-13), возвращает JSON
2. map_layer_splitter.py — парсер PPTX отделяет фон от объектов
3. map_background.py — замена/улучшение подложки по типу
4. map_objects_redesign.py — перекраска линий/фигур/текста в brand colors
5. map_assembler.py — сборка: новая подложка + редизайнутые объекты → PPTX

### 13 типов карт и стратегии:
| Тип | Описание | Стратегия подложки |
|-----|----------|-------------------|
| 1 | Спутник чистый + PPTX-объекты | НБ2 улучшение |
| 2 | Спутник с надписями + PPTX-объекты | Gemini OCR → координаты → Mapbox |
| 3 | Карта мира с выделенными странами | SVG из assets/world_map + покраска |
| 4 | Страна/город по районам | OSM boundaries → SVG, fallback Gemini сегментация |
| 5 | Обычная карта + PPTX-объекты | Gemini OCR → координаты → Mapbox |
| 6 | Сложный растр (всё впечатано) | Fallback: НБ2 улучшение |
| 7 | Схематическая (метро, этажи) | Оставить/НБ2, редизайн цветов |
| 8 | Маршрут А→Б | Как тип 2/5, маршрут = PPTX-объект |
| 9 | Тепловая карта (heatmap) | Заменить подложку, перекрасить зоны |
| 10 | Карта с фото/иконками | Заменить подложку, редизайн объектов |
| 11 | Скриншот Google Maps/2GIS/Yandex | Gemini OCR → координаты → Mapbox |
| 12 | Инфографика с мини-картой | Определить подтип, применить его стратегию |
| 13 | Тематическая/статистическая | Gemini читает данные → перерисовка программно |

### Проверенные инструменты для карт:
- HSV-фильтрация (S≥80) на оригинальную карту — отделяет яркие маршруты от бледной подложки
- Skan (skeleton analysis) — превращает скелет в упорядоченные ветки с координатами
- Douglas-Peucker (cv2.approxPolyDP, epsilon=0.02) — упрощение до 10-20 точек на ветку
- Gemini Vision OCR — читает текст с карты для определения координат
- Mapbox Static API — получение чистой подложки по координатам + zoom
- Natural Earth GeoJSON → SVG — карты мира/стран с границами (public domain)

### Что НЕ работает для карт (проверено):
- Nano Banana 2 для извлечения слоёв — галлюцинирует, каждый раз разный результат
- Gemini координаты [x%, y%] маршрутов — ошибка 5-15%
- Gemini геолокация спутника без текста — 0% точности
- SAM 2 — не подходит для тонких линий
- Морфологическое закрытие (MORPH_CLOSE) — деформирует линии (критично для техиллюстраций)
- EasyOCR — таймаут загрузки модели на Windows
- keras-ocr — требует Visual C++ Build Tools
- Potrace — обводит толстые линии двойным контуром (для заливок ОК, для линий нет)
- Вычитание фонов (dual-image subtraction) — невозможно совместить пиксель-в-пиксель

### Gemini 2.5+ сегментация (перспективная фича):
Gemini 2.5+ и Gemini 3 умеют возвращать JSON с bounding box + base64 PNG маска для каждого объекта.
Промпт: "Give segmentation masks for all route lines and zone areas on this map."
Потенциально заменяет HSV+Skan для сложных растров (тип 6).
Статус: НЕ ТЕСТИРОВАНО, запланировано.

## Утверждённый пайплайн (редизайн)
1. Клиент загружает PPTX + акцентный цвет + режим текста (editable/locked)
2. Парсер (pptxtojson, JS) → структура.json (100% точность, без OCR)
3. Конвертер → slide_X.jpg
4. Vision-классификатор (Gemini 2.5 Flash) → классификация изображений, пропорции, качество, растянутость
5. Центральный мозг (Gemini 3.1 Pro) — арт-директор:
   - Уровень 1: саммари всей презы → стратегия + тип презентации + стилевой пресет
   - Правило "30 секунд спикера" (Chain of Thought)
   - Надстройка по типу: выступление / рассылка / техотчёт / обучение
   - Уровень 2: пачки по 5-10 слайдов → бриф для каждого слайда
6. Подборщик (Gemini 2.5 Flash) → поиск few-shot примера в catalog.json
7. Тестер (Python) → если жёсткий шаблон подходит → подстановка ($0)
8. Если нет → Дизайнер-верстальщик (Gemini 3.1 Flash) генерирует SVG по брифу
9. Реверс-инжиниринг сложных элементов:
   - Блок-схемы → Vision → JSON-граф → Graphviz → SVG → PPT Master
   - Диаграммы → Vision → CSV → Matplotlib/python-pptx → нативный график
   - Карты → Map Redesign Pipeline v2.0 (13 типов, 5 модулей) # UPDATED
10. Художник (Nano Banana 2) → AI-изображения + улучшение спутника
11. Постобработка SVG (finalize_svg.py)
12. PPT Master (svg_to_pptx.py) → PPTX
13. Постобработка PPTX (python-pptx: паддинги, скругления, тени)
14. Инспектор (Gemini 2.5 Flash) → проверка по good_slide_rules.md (max 2 попытки)
15. Клиент скачивает результат

## Утверждённый пайплайн (генерация с нуля) — дешевле
1. Клиент загружает текст/документ + акцентный цвет
2. Центральный мозг → анализ + стратегия + план слайдов

## Design System v2.0

### Три документа (разделение ответственности):
| Документ | Для кого | Что делает |
|----------|----------|------------|
| layout_code.md | Мозг + Дизайнер | Как располагать элементы (компоновка) |

| good_slide_rules.md | Инспектор | Чеклист проверки качества |

### Промпт Мозга (универсальный, Chain of Thought):
"Представь: ты спикер перед аудиторией, 30 секунд на слайд.
1) Что я хочу чтобы они ПОНЯЛИ?
2) Что увидеть ПЕРВЫМ, ВТОРЫМ, ТРЕТЬИМ?
3) Что можно УБРАТЬ?"

### Надстройки по типу презентации:
- Выступление/питч: минимум текста, одна мысль, цифры огромные
- Рассылка/КП: понятно без спикера, логика убеждения
- Техотчёт/госсектор: полнота данных, таблицы ОК, статусы
- Обучение: пошагово, нумерация, один шаг = один блок

### Размеры (viewBox 1280×720):
- Слайд: 33.87cm × 19.05cm
- Margins: 47px (1.25cm) со всех сторон
- Рабочая область: 1186×626px (x: 47–1233, y: 47–673)
